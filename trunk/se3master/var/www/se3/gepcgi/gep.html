Fichiers gep nécessaires

Le schéma ci-dessous représente les fichiers nécessaires et les informations qui en sont extraites :

  • du fichier f_ele.dbf sont extraits le prénom, le nom, la date de naissance, le sexe et la classe des élèves ;
  • de f_eag.dbf, les liens entre modules d'enseignement et, groupes ou classes ;
  • de f_gro.dbf, le nom des groupes et leur code ;
  • de f_div.dbf, le nom des classes propres à l'établissement, leur code et les identifiants des professeurs principaux ;
  • de f_men.dbf, les intitulés des différentes modules d'enseignement (ou cours :) et les liens qui les relient aux matières, aux groupes d'élèves et aux professeurs associés ;
  • de f_tmt.dbf, les noms des matières associées aux cours, propres à l'établissement ;
  • de f_wind.dbf, le nom, le prénom, la date de naissance, le sexe et le code discipline des professeurs.
  • de a_ind.dbf, le nom, le prénom, la date de naissance, le sexe et le code discipline des personnels administratifs.
  • et optionnellement de f_uid.txt, l'identifiant (nom de login ou uid) des utilisateurs.

    • Format du fichier optionnel f_uid.txt

    Entrée type : 01561|Celine.Bigeon

    Descriptif des différents champs :

    • Numéro unique de l'élève ou du professeur dans l'établissement (ELENOET ou NUMIND)
    • Identifiant
    Rmq1. S'il n'y a pas de ligne définie dans ce fichier pour un utilisateur, le nom de login est généré à partir de son prénom et de son nom selon la valeur du paramètre uidPolicy.
    Rmq2. Attention ! : Le numéro unique des enseignants doit impérativement être préfixé par un P.

Importation

Préfixe

Si votre établissement est composé de plusieurs entités, comme un lycée professionnel et un lycée technique par exemple, vous diposez de plusieurs bases gep.

il faut donc spécifier un suffixe pour chacune de ces bases afin de différencier les importations nécessairement séparées..

Si vous ne disposez que d'une seule base de fichiers gep, laissez ce champ vide.

Importation annuelle

L'importation annuelle nécessite une préparation en profondeur de l'annuaire pour l'intégration des nouveaux élèves et la régénération des nouveaux groupes de 'Classe', 'Equipe', 'Cours' et 'Matiere'.

Le fait de cocher la case ci-dessous permettra la mise en place automatique de cette préparation.
